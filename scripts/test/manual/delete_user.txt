提供されたコードやドキュメントに基づくと、このシステムにはユーザー自身や管理者がWeb画面（UI）やAPI経由で「特定のユーザー情報をリセット（初期化）する」ための機能は実装されていません。

そのため、ユーザーに関する情報（会話履歴やプロファイル情報）をリセットするには、**データベース（MySQL）に直接接続し、該当するデータを削除する手動操作**が必要です。

具体的な手順は以下の通りです。

### 1\. データベースへの接続

システムに含まれるスクリプト `scripts/test/manual/db_connect.sh` を参考に、Dockerコンテナ内のMySQLに接続します。

ターミナルで以下のコマンドを実行します：

```bash
docker exec -it ai-agent-playground-101-db-1 mysql --default-character-set=utf8 -u me -p mydb
```

  * **パスワード**: `me`

※ コンテナ名 `ai-agent-playground-101-db-1` は環境によって異なる場合があります（`docker ps` で確認可能です）。

### 2\. ユーザーデータの削除

MySQLに接続後、以下のSQLを実行して特定のユーザーを削除します。

```sql
-- ユーザーIDまたはLINEユーザーIDを指定して削除
DELETE FROM users WHERE line_user_id = '対象のLINEユーザーID';
-- または
DELETE FROM users WHERE id = '対象のユーザーID';
-- すべて削除する場合は
DELETE FROM users;
```

**解説:**

  * `users` テーブルからレコードを削除すると、設定されている外部キー制約（`ON DELETE CASCADE`）により、関連する以下のテーブルのデータも**自動的に全て削除**されます。
      * `user_states`（住民プロファイル、サービスニーズなどの現状分析データ）
      * `user_messages`（会話履歴）
      * `user_message_analyses`（分析ログ）

### 3\. 全データの完全リセット（開発環境向け）

もし特定のユーザーではなく、システム全体の全ユーザーデータをリセットしたい場合は、Dockerのボリュームデータを削除する方法があります。

`docker-compose.yaml` の設定により、MySQLのデータはホスト側の `./mysql/data` ディレクトリに保存されています。

```bash
# コンテナを停止・削除
docker compose down

# データディレクトリの中身を削除（注意：全データが消えます）
rm -rf mysql/data/*

# コンテナを再起動
docker compose up -d
```

これにより、データベースが初期状態（空の状態）で再構築されます。

### 補足：APIやUIの実装状況

  * 現在のAPIには、サービスカタログをリセットする `/api/v1/service-catalog/reset` はありますが、ユーザー情報をリセットするエンドポイントは存在しません。
  * ユーザーの状態（`user_states`）は会話のたびにデータベースから読み込まれ、更新される仕組みになっており、明示的に削除しない限り保持され続けます。
