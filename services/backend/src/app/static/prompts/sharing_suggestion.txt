## タスク

この仮説をチームに共有することを提案するために、以下を行ってください：

### 1. 共有価値の評価
- この仮説がチームメンバーにとって有益な理由を説明
- 想定される受益者（どの部署・役割の人に役立つか）
- 共有のタイミングが適切かどうかの判断

### 2. 匿名化ドラフトの作成
- 個人を特定できる情報を除去
- 汎用的な表現に変換
- 本質的な知見は維持

### 3. 共有メッセージの生成
- ユーザーへの提案文（丁寧かつ押し付けがましくない）
- 共有のメリットの説明

## 出力形式

以下のJSON形式で出力してください：

```json
{{
  "should_suggest": true,
  "suggestion_reason": "共有を提案する理由の説明",
  "target_audience": ["想定される受益者のリスト"],
  "anonymized_draft": {{
    "statement": "匿名化された仮説の本文",
    "context": "一般化された背景説明",
    "key_insight": "核心となる知見"
  }},
  "user_message": "ユーザーへの提案メッセージ",
  "sharing_benefits": ["共有することのメリットリスト"]
}}
```

## 提案を控えるケース

以下の場合は should_suggest を false にしてください：
- 品質スコアが低い（overall_score < 0.5）
- 個人的すぎて一般化が難しい
- 検証状況が「FAILED」で教訓としての価値も低い場合
- 機密性の高い情報を含む可能性がある場合
